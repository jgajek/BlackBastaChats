[19:18:29] zdays: # Unautheticated root RCE on JunOS

JunOS is an operating system based on FreeBSD developed by Juniper networks to run on firewall/vpn devices. This OS manages the device and is responsible for operating services. The device is secured in multiple ways like using read-only file systems for packages/binaries in the system as well as veri-exec which disables executing unsigned or unkown binaries.

It also has a web management interface that runs on Embedthis Appweb web server and has php scripts executed in cgi mode for dynamic content. The execution of PHP scripts as cgi using appweb is where the first issue lies.

Appweb executes CGI scripts/binaries using the CGI/1.1 standard, But it messes up when exporting environment variables for said scripts/cgis. This appears to be fixed in the latest version of the web server but the version JunOS uses are affected. Also, I don't expect JunOS will update the appweb version any time soon since there's a lot of custom functionality that they have added to it. Regardless the appweb version in JunOS 12.1 (Released 2015) to JunOS 23.1R1 (Released 2023) are vulnerable.


[19:18:29] zdays: The vulnerability is that request parameters sent via POST or GET are exported to environment variables. This was used before to exploit CGI scripts using __LD_PRELOAD__ and loading the POST data as a library. But JunOS uses PHP and the __/proc__ filesystem does not work for exploitation on FreeBSD, Not only that veri-exec will prevent the loading of unsigned and unknown libraries.

To exploit the bug, I used a specific behaviour of php, I will not dive into the php source code and how I discovered this behaviour to keep the report breif but basically PHP executes a script or path specified in __PATH_TRANSLATED__ if the __REDIRECT_URL__ variable is set, Using this we can execute PHP scripts outside the webroot, The next step is to find a way to write a piece of PHP code in a path we can access and execute it.
